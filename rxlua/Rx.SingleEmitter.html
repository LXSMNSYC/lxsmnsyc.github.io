SingleEmitter

Rx. SingleEmitter

Abstraction over an Rx.SingleObserver that allows associating a resource with it.

Calling onSuccess(Object) multiple times has no effect. Calling onError(Throwable) multiple times or after onSuccess will route the exception into the global error handler via HostError.

The emitter allows the registration of a single resource, in the form of a Rx.Disposable via setDisposable(Disposable). The emitter implementations will dispose/cancel this instance when the downstream cancels the flow or after the event generator logic calls onSuccess(Object) or onError(Throwable) succeeds.

Only one Disposable object can be associated with the emitter at a time. Calling setDisposable method will dispose any previous object. If there is a need for handling multiple resources, one can create a Rx.CompositeDisposable and associate that with the emitter instead.
